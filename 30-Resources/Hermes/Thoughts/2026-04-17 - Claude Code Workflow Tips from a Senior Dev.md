## Source thought

"Claude Code workflow tips after 6 months of daily use (from a senior dev)

I’ve been using Claude Code daily for months now (I’m a senior full-stack dev). Here’s the workflow that's made me genuinely productive after a lot of trial and error.

The basics that changed how I work:

- Use \"plan\" mode for anything complex. Before Claude writes a single line, let it lay out its approach.
- Only ask for the first step. If you say \"implement the whole feature\", it can go off the rails. Ask for step one, review, then continue.
- Use the preview.
- Don’t fix bugs yourself, let Claude fix them so it builds better context on your codebase.
- Run /simplify before review because Claude tends to over-engineer.
- Do a retro at the end of each session: ask \"what did you learn during this session?\" and save the output for institutional knowledge."

## Why this is useful

This is a practical control loop for AI-assisted coding:
1) plan first,
2) execute in small increments,
3) review previews,
4) let the model self-correct,
5) simplify before final review,
6) capture learnings.

Good reminder: throughput comes from tighter feedback loops, not bigger prompts.
